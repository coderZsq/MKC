## 技术要点

- README 第一屏展示项目定位、Demo 截图或架构概览，不写成纯 TODO 列表。
- 架构图可用 Mermaid 源码入库，必要时导出图片。
- 快速开始命令按“准备环境 -> 启动依赖 -> 启动服务 -> 打开客户端 -> 运行测试”组织。
- 部署文档与 S5-8 保持一致，避免 README 内重复大量 K8s 细节。

## 阻塞风险

| 风险 | 影响 | 应对 |
|---|---|---|
| 文档与真实命令不一致 | 新用户无法运行 | 每条命令本地验证或标注前置条件 |
| 架构图过度简化 | 无法体现项目复杂度 | 同时提供逻辑架构与请求链路图 |
| 泄露配置 | 安全风险 | 使用 `.env.example` 和占位符 |
